# OpenClaw Usage Rules for This Vault

This vault is a curated knowledge layer, not a replacement for code, schemas, prompts, or live-order confirmation rules.

## Rules

1. Never treat a note as an automatic live-trading instruction.
2. Never infer order placement authorization from wiki content.
3. Prefer raw evidence and structured records for final audit metrics.
4. Use notes to identify patterns, hypotheses, lessons, and links to evidence.
5. Patch prompts/tools/docs only when evidence is repeated, clear, or safety-critical.
6. Keep single-trade outcomes as observations unless confirmed by later cases.
7. If notes conflict with active project files, active project files win until deliberately patched.

## Evidence chain

Raw data/export → structured audit record → Obsidian distilled note → approved workflow patch

## Good note types

- setup postmortem
- recurring failure pattern
- liquidity/orderability lesson
- symbol-specific quirk
- prompt/tool patch rationale
- unresolved research question
