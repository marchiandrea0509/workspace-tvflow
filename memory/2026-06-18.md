## ASML deep-analysis first-shot format failure / marker-leak fix
- Andrea correctly flagged that the 2026-06-18 ASMLUSDT.P reply still had bad first-shot formatting: the assistant pasted `VERBATIM_DEEP_ANALYSIS_CHUNK ...` marker text into the visible Discord answer and only delivered the beginning of the chunk set, making the output look like internal tooling rather than a clean report.
- Root cause: `scripts/finalize_deep_analysis_delivery.py` wrote control markers directly into user-visible `chunk_*.md` files while also instructing the assistant to send those chunks verbatim. The anti-regression guard therefore created a new delivery bug: following the instruction exactly leaked internal metadata. A second issue is that the model still had to manually paste many chunks in one final answer, so the completion path could truncate/stop early instead of delivering all chunks cleanly.
- Patch applied: finalizer now writes clean user-visible chunk files with no `VERBATIM_DEEP_ANALYSIS_CHUNK` marker, keeps delivery instructions in `manifest.json`, validates that chunks do not contain internal marker text, and prints `send clean chunk_*.md content verbatim`. Validation: `python -m py_compile scripts\finalize_deep_analysis_delivery.py` and rerun finalizer for `reports\deep_analysis\2026-06-18_ASMLUSDT_deep_analysis.md` passed; regenerated `chunk_01.md` starts directly with the report title.
- Future rule: deep-analysis delivery chunks are user-visible artifacts. Never put internal control text in them. If multiple chunks are needed, prefer clean chunks and send in order; do not combine marker/instruction metadata with the actual report content.
