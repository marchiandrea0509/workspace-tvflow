
## BTCUSDT HYBRID VOCO watchdog backgrounded-output fix
- Andrea received a misleading Discord failure from BTCUSDT HYBRID VOCO B/C2 watchdog: `watchdog command did not return a completed JSON result before the tool session backgrounded, so status could not be verified`.
- Root cause: cron `f5f978aa-e4fc-46e3-8be7-e0ae7fcec0a1` allowed only `exec`, and the prompt did not force a long enough `exec.yieldMs`. When the watchdog command exceeded the default exec wait, it backgrounded; because `process` was not allowed in that isolated cron run, the agent could not poll the session and incorrectly reported a blocker. This was not an exchange/order failure.
- Patched the cron payload: tools now allow `exec,process`; timeoutSeconds increased to `240`; prompt explicitly says to run the read-only watchdog command with long `exec` wait (`yieldMs=180000`, `timeout=180`) and to `process.poll` if it still backgrounds. It must not report failure merely because the command backgrounded; only nonzero exit, unrecoverable JSON after polling, or real Discord send failure counts as failure.
- Direct verification command completed with JSON status `CHECKED_ALREADY`; no trigger, no live order placed/cancelled by watchdog. BTC B ladder remains live and C2 virtual watchdog remains armed.
