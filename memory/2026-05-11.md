## Deep-analysis v2 ladder and screenshot delivery patch
- User reported the deep-analysis tool was too strict, often producing only one or two legs, and that screenshots still were not delivered.
- Root cause found in `scripts/build_deep_analysis_packet_v2.py`: ladder geometry used latest closed 4H close as the current price even when live ticker had moved materially, so fast moves near highs made pullback levels appear too shallow/deep. LC inactive/hot-price logic also capped meaningful deeper structure at ~6 ATR, which often collapsed the ladder to one/two entries. Stop logic could anchor to the next far lower cluster, creating too-deep swing stops like the INTC `106.27` case.
- Patched builder: when ticker differs from latest closed 4H by >0.15%, ladder geometry uses ticker/current price while closed candles remain primary analysis truth; LC/DIP max depth expanded to allow meaningful 4H/EMA structure up to 10 ATR / 8%; spacing relaxed slightly; stop logic now uses immediate 4H-structure invalidation when the next lower cluster is far below the selected ladder instead of automatically widening to a swing stop. Added `ladder_price_reference` diagnostics to analysis summaries.
- Smoke-tested patched builder on NVDA without TV capture under `reports/deep_analysis_packets_v2_patchtest/20260511_160025_NVDAUSDT/`; patched output produced 3 candidate legs instead of 1, but correctly warned about weak L1 R:R and margin cap, so final judgment still needs analyst filtering.
