- Andrea approved making the new generic OHLCV-first deep-analysis packet workflow the default, while keeping the old analyser/scripts/prompts for early comparison and reference.

- Current screener version context for this new deep-analysis workflow is `OC Hybrid Edge Screener v11.6.x`.
- For TradingView evidence, prefer the existing Playwright/browser capture/export path unless TradingView Desktop MCP proves more robust or cheaper. Robustness and runtime cost drive the choice.

## V2 ladder-selection bug diagnosed and patched
- Andrea correctly noted old-v1 MRVL levels looked more reasonable than v2: v2 produced L1 `170.36` and L2 `169.82` almost at market / same cluster, then jumped to L3 `162.09`.
- Root cause: v2 ladder builder used fixed ATR depth buckets and always selected nearest supports inside shallow buckets, so it over-selected 1H near-market noise. It did not sufficiently use screener action/window fields (`W01 LC ActionScore=0`, `W02 LC WindowActive=0`, `W03 LC EntryState=0`) to suppress shallow LC legs. It also hid weak natural R:R by projecting optimistic targets instead of showing nearby resistance.
- Patched `scripts/build_deep_analysis_packet_v2.py`: `build_ladder` now accepts `screener_data`; LC/DIP ladders use screener action/window context, reject too-shallow near-market levels when price is hot/near resistance or LC action is inactive, cluster/select meaningful structural zones, prefer 4H pivots/EMA50/1H EMA200, set stops beyond a lower structural cluster rather than just under the nearest pivot, and expose weak natural R:R instead of forcing projected targets.
- Smoke retest on MRVL with same fresh Screener CSV produced improved v2 ladder: L1 `167.08` (4H EMA50) qty `2.54` SL `157.26` TP `175.99` R:R `0.91` flagged weak; L2 `164.96` qty `4.54` SL `157.26` TP `175.78` R:R `1.41`; L3 `162.09` qty `8.28` SL `157.26` TP `171.05` R:R `1.86`; estimated 4x margin `628.85`, feasible under `1500`. Warnings now explicitly say LC action/window inactive and shallow legs should be avoided.
