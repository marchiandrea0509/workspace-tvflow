# 2026-05-10

## Deep-analysis starter pack decisions
- Andrea approved making the new generic OHLCV-first deep-analysis packet workflow the default, while keeping the old analyser/scripts/prompts for early comparison and reference.
- Initial v2 design used `1500` as max total notional, but Andrea corrected this later: the `1500` cap in deep-analysis trade tickets now means **max margin at planned leverage**, not max total notional.
- Live execution is excluded from deep-analysis rounds. Deep analysis may propose a ticket only; Andrea will always ask separately and explicitly for any live execution.
- Remove hardcoded screener score eligibility such as `score >=70`. Andrea will point to a selected symbol from the screener; the deep analysis should still target **100 USDT planned risk** by default.
- If structure, R:R, freshness, or notional constraints are weak/invalid, report strong warnings or WAIT/NO_TRADE; do not silently reduce the target risk just because confidence is lower.
- Ladder generation is especially important: entries must balance R:R with realistic expected pullback depth and meaningful structure. Do not reject LC/DIP ladders merely because price is near resistance or RSI is high; use stronger no-ladder reasons such as CHoCH, degraded trend, high SL-hit probability, stale data, liquidity/fee issue, or objectively poor R:R.
- Current screener version context for this new deep-analysis workflow is `OC Hybrid Edge Screener v11.6.x`.
- For TradingView evidence, prefer the existing Playwright/browser capture/export path unless TradingView Desktop MCP proves more robust or cheaper. Robustness and runtime cost drive the choice.
